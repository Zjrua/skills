# 根据数据规模反推算法复杂度

竞赛/LeetCode 时间限制一般 1~2 秒。C++ 约 10^8 次运算/秒，Python 约 10^7 次（慢 5~10 倍）。

## 速查表

| n 的规模 | 可接受复杂度 | 典型算法 |
|----------|-------------|----------|
| n ≤ 10 | O(n!)、O(2^n × n²) | 全排列暴力、状态压缩 |
| n ≤ 20 | O(2^n) | 状压DP、子集枚举 |
| n ≤ 100 | O(n³) | Floyd、区间DP |
| n ≤ 500 | O(n³) 勉强 | 三维DP |
| n ≤ 1000 | O(n²) | 双重循环、背包DP |
| n ≤ 10^4 | O(n²) | 冒泡排序都行 |
| n ≤ 10^5~10^6 | O(n log n) | 排序、分治、堆 |
| n ≤ 10^7 | O(n) | 线性扫描、哈希表 |
| n ≥ 10^9 | O(log n) 或 O(√n) | 二分、数学公式 |

## Python 注意

Python 比 C++ 慢 5~10 倍，要降一档：
- n ≤ 10^4 → O(n²) 勉强能过
- n ≤ 10^5 → O(n log n)
- n ≤ 10^6 → O(n) 才稳

## 空间复杂度

- n ≤ 10^5 → 可以开 O(n) 的数组/哈希表
- n ≤ 10^7 → O(n) 要小心内存（int数组约 40MB）
- n ≥ 10^9 → 只能 O(1) 或 O(log n)

## 实战口诀

拿到题先看 n，秒选复杂度区间，再想算法。
